# LLMs.txt

Tau publishes machine-readable documentation in the [llms.txt](https://llmstxt.org/) style so assistants can index Runtime docs, answer integration questions, and stay aligned with published APIs.

## Runtime-only exports

Use these endpoints when you want **only** the Runtime documentation (Getting Started, Guides, Concepts, API Reference), not the Editor docs.

| Endpoint                                                     | Purpose                                                                             |
| ------------------------------------------------------------ | ----------------------------------------------------------------------------------- |
| [`/docs/runtime/llms.txt`](/docs/runtime/llms.txt)           | Index of every Runtime page with titles, URLs, and frontmatter `description` lines. |
| [`/docs/runtime/llms-full.txt`](/docs/runtime/llms-full.txt) | Full processed markdown for every Runtime page, concatenated in one file.           |

For a smaller context window, start with `llms.txt`, then fetch individual pages (see below) as needed.

## Whole-site exports

The same machinery also covers **all** documentation trees (Runtime and Editor):

| Endpoint                           | Purpose                                |
| ---------------------------------- | -------------------------------------- |
| [`/llms.txt`](/llms.txt)           | Site-wide index (grouped by doc root). |
| [`/llms-full.txt`](/llms-full.txt) | Site-wide concatenated markdown.       |

## Per-page markdown

Any docs URL can be turned into raw markdown for a single page by appending `.mdx` to the path (the app redirects to the plain-text route). Example: [`/docs/runtime/getting-started/installation.mdx`](/docs/runtime/getting-started/installation.mdx).

## Quick setup for AI tools

**Cursor:** Open **Cursor Settings → Indexing & Docs → Add Doc**, and paste `https://tau.new/docs/runtime/llms.txt` (or your deployment origin plus `/docs/runtime/llms.txt`).

**Chat clients:** Paste the same URL into the assistant’s “add documentation” or context URL field if supported.

**Command line:**

```bash
curl -sSf https://tau.new/docs/runtime/llms.txt | head
```

Replace the origin when working locally or on staging (for example `http://localhost:5173` if that is where your docs dev server runs).
